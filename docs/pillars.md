# Design Pillars

## 1. Immediate Loop Clarity

A new player should be able to understand the main loop quickly:
heal -> plant -> break -> buy.

## 2. Shared-World Fairness

The world may be shared, but rewards and ownership must stay clear and server-authoritative.

## 3. Folktale Flavor Without System Bloat

The Heungbu/Nolbu identity should come through world tone, loop verbs, and pressure beats without reopening heavy role, house, or branching systems by default.

## 4. Additive Evolution

The repo should evolve through small shippable slices that preserve playability, not through speculative large rewrites.

## 5. Truthful Operations

Docs, agent guidance, slice tracking, and validation steps must describe the game and repo that actually exist today.
