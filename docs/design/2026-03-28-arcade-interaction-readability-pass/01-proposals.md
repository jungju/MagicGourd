# Proposals

## 1. Contextual Next-Action Readability Pass
- **Goal:** make the HUD always answer "what should I do now?" using current player state.
- **Core idea:** the status/tip area changes based on seeds, money, slowed state, and nearby loop phase.
- **Why it helps:** strongest gain for first-play clarity and repeated-loop flow.
- **Risk:** could become naggy if copy changes too often.

## 2. Upgrade Comparison Polish
- **Goal:** make plaza upgrades readable as fast choices instead of three text rows.
- **Core idea:** stronger visual hierarchy for current level, next benefit, affordability, and maxed state.
- **Why it helps:** improves spending confidence and reduces "which one was this again?" friction.
- **Risk:** if overdesigned, it bloats the compact HUD.

## 3. Feedback Tone / Event Precision Pass
- **Goal:** make successful and failed actions feel more distinct.
- **Core idea:** keep the current feedback banner, but make copy/tone/state mapping more specific by event type.
- **Why it helps:** better reward feel and clearer cause/effect without new effects systems.
- **Risk:** feedback can still be missed if the banner remains too transient.

## 4. Prompt Naming and World Copy Cleanup
- **Goal:** make prompts, zone labels, and action words more internally consistent.
- **Core idea:** standardize verbs and noun phrases so every interactable reads quickly and predictably.
- **Why it helps:** low-risk polish for interaction confidence.
- **Risk:** useful but narrower than broader HUD flow improvements.
