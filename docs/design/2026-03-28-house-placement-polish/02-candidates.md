# Candidate Set

## Candidate 1 - Explicit House Placement Contract
- Core loop: define upright correction + yard-facing orientation + bottom anchoring for each house, then place imported and fallback versions through the same contract.
- Theme role: strengthens the Heungbu vs Nolbu contrast in the first visual read.
- Expected payoff: cleaner village composition with low future confusion.

## Candidate 2 - Fallback-First Final Houses
- Core loop: stop trusting imported houses, improve the fallback houses, and treat them as the intended shipped look.
- Theme role: keeps the story legible even if assets are unreliable.
- Expected payoff: robust but visually less distinctive.

## Candidate 3 - Placement Calibration Layer
- Core loop: create a reusable calibration/debug layer around pads, pivots, footprint extents, and facing arrows.
- Theme role: indirectly supports world readability.
- Expected payoff: strongest long-term tool, but heavier than the current problem may justify.
