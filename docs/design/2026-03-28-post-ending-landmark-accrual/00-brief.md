# Feature Brief

## Name
Post-Ending Landmark Accrual

## Problem
The post-ending layer now has branch mood, repeatable errands, milestone escalation, and a support-point rotation design. What it still lacks is a strong sense that repeated help leaves visible traces in the village. Players can perform meaningful post-ending actions, but the world does not yet accumulate that meaning in a way that reads at a glance.

## Desired Player Experience
After several post-ending hand-ins, the player should notice that each ending slowly authors the village:
- restoration should look more supported, stocked, and collectively cared for
- tribute should look more regulated, counted, and normalized

The player should feel that replay is building a state, not only triggering another prompt.

## Why Now
This is a strong follow-on after support-point rotation because:
- current code already tracks branch errand counts
- runtime signs and small accent props already exist
- support rotation spreads replay across multiple landmarks, which creates natural places for light visual accumulation
- it adds payoff without opening a new quest chapter or economy layer

## Constraints
- Roblox constraints: use primitive/runtime-authored props and billboard text; avoid asset-heavy staging
- Rojo constraints: keep the system data-driven and server-authored in existing village runtime patterns
- Narrative constraints: reflect Heungbu-style communal recovery vs Nolbu-style normalized tribute, without rewriting the ending
- Scope constraints: cosmetic/stateful polish only; no new quest stage, no new resource type, no complex persistence tree

## Existing Systems Touched
- COMPLETE ending free play
- branch-specific errand counters and milestone thresholds
- support-point signs / hints / prompt copy
- ending accent props and runtime village signage

## Success Signal
Within 3-6 post-ending errands, the player can tell from the village landmarks that their chosen ending is continuing to shape the world, even before reading dialogue.
