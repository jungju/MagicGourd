# Brief

## Feature
Post-ending rare event variants

## Why now
The post-ending village loop is structurally playable, but repeated hand-ins can start to feel too even. The next safe increment is not a new questline; it is a tiny layer of rare, branch-specific payoff moments that occasionally make repetition feel newly noticed.

## Goal
Add low-frequency special post-ending errand moments that:
- trigger only during the existing `COMPLETE` replay loop
- reinforce restoration vs tribute tone
- reuse existing dialogue/effect flow
- avoid adding a new progression tree

## Scope
- one lightweight server-side rare-event selector
- branch-specific event pools
- activation on select post-ending errand milestones
- small bonus flavor rewards only where already economy-safe

## Out of scope
- new NPC schedules
- new UI panels
- persistent branching subquests
- heavy randomness-dependent economy systems
