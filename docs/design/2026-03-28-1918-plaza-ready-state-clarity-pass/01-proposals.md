# Proposals

## Proposal A — Ready-state ownership contract
Define one shared presentation contract for upgrade rows and nearby plaza support so affordability has a single obvious read.

## Proposal B — Threshold-crossing flash handoff
Add a stronger but still brief handoff rule right after a payout crosses a threshold, so the newly-ready buy is easier to notice.

## Proposal C — Plaza comparison quieting only
Do not emphasize any one ready option much; instead, mostly quiet non-ready/max rows so the readable option emerges by subtraction.

## Proposal D — Named recommendation bias
When multiple upgrades are affordable, pick one recommended buy and push it harder than the rest.

### Initial note
A/B/C fit refinement mode.
D risks over-directing the player and creeping into recommendation-system territory.
