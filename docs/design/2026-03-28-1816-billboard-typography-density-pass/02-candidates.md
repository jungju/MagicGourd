# Candidate Set

After compression, keep these three candidates:

## Candidate A — Hierarchy contract only
A small pass that defines title/body emphasis and copy budgets per sign class.

### Pros
- cheap and focused
- directly targets the current density problem

### Cons
- does not say enough about `AlwaysOnTop` and depth behavior
- may leave plaza-specific overlap ambiguities under-specified

## Candidate B — Plaza-only quieting
Treat Central Plaza as the only urgent density hotspot.

### Pros
- strongest payoff per line changed in the busiest area
- easy to implement and validate

### Cons
- too narrow; leaves spawn/lane/world sign grammar partially implicit
- risks one-off tuning instead of a reusable rule

## Candidate C — Full billboard presentation contract
Define typography, density, and support-sign priority rules across all current billboard classes.

### Pros
- closes the remaining readability gap at the presentation layer
- gives Studio validation a clearer tie-breaker when signs feel noisy
- improves consistency without expanding scope

### Cons
- slightly broader authoring doc than a plaza-only pass
- must stay disciplined to avoid turning into a full UI redesign

## Adoptable Range
Candidate C is preferred if it stays small and strictly limited to existing billboard usage.
