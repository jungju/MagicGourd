# Brief

## Refinement Target
Current arcade play already works, but the player-facing language is still split across multiple vocabularies:
- button says `Use Seed`
- pumpkin prompt says `Hit`
- upgrade prompt says `Upgrade`
- zone signs describe loops in longer sentences
- feedback text alternates between `planted`, `ready`, `broken`, `cash out`, and generic status lines

This is a polish problem, not a mechanics problem. The loop is short enough that wording inconsistency creates more friction than feature absence.

## Why This Matters Now
The project is in refinement mode. That means the highest-value work is often about **clarity and feel inside the current structure**, not expansion. When the same action is described with different verbs depending on location, the game feels less finished than it actually is.

## Target Outcome
Create a precise copy-and-signage pass that makes the existing loop read the same way everywhere:
- HUD guidance
- action button
- prompts
- world sign billboards
- feedback banners
- world popups

## Scope Boundary
In scope:
- naming consistency
- copy hierarchy
- sign body tightening
- feedback bucket wording
- interaction label rules

Out of scope:
- new mechanics
- new progression systems
- new world props beyond what is needed for text placement
- new tutorial UI flows
- narrative expansion
