# Feature Brief

## Name
MagicGourd Design Agent Workflow

## Problem
지금까지 기능 확장이 빠르게 진행되었지만, 아이디어 제안/후보 비교/채택 근거/정교한 설계/검증 단계가 프로젝트 안에 명시적으로 남아 있지 않았다.

## Desired Player Experience
플레이어가 만나는 기능이 더 일관되고, 흥부전 서사와 시스템 확장이 자연스럽게 이어지는 게임이 된다.

## Why Now
이제 MagicGourd는 초기 플레이 루프가 생겼기 때문에, 다음 기능부터는 품질 높은 설계 체계를 두는 편이 전체 완성도를 크게 올린다.

## Constraints
- Roblox constraints: Studio 수동 확인이 필요한 요소가 있음
- Rojo constraints: 코드/모델/문서 중심으로 구조를 유지해야 함
- Narrative constraints: 흥부전 메인 테마를 흐리지 않아야 함
- Scope constraints: 설계 체계가 너무 무거워지면 실제 개발 속도를 해칠 수 있음

## Existing Systems Touched
- project documentation
- future feature planning workflow

## Success Signal
다음 기능 설계부터 proposal -> candidate -> analysis -> adopted design -> verification 흐름을 실제로 남길 수 있다.
