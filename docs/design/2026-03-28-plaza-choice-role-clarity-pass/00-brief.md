# Brief

## Refinement Goal
Improve **plaza upgrade choice clarity** inside the current arcade loop so players can tell, at a glance, what each pad changes and when it is worth buying.

## Why this matters now
The live loop is already compact and functional:
- heal a swallow
- plant a pumpkin
- break for money
- buy an upgrade

The remaining weakness is not missing content. It is that the **buy step is still less self-explanatory than the other three steps**.

Current strengths already in place:
- prompt distance is tighter than village/pumpkin interactions
- HUD rows show affordability and next-level benefit
- pad billboards have short purpose text
- purchase feedback is now plaza-anchored

Current completion gap:
- the three upgrades still risk reading as three similar pads instead of three distinct pacing choices
- the player can understand affordability before understanding role difference
- the buy moment can feel like account management instead of a clean route decision

## Intended outcome
Without adding new mechanics, make each plaza option read as a distinct answer to a distinct loop question:
- **Footwork** = get around faster
- **Quick Hands** = finish pumpkins faster
- **Blessing** = get more seeds from healing

## Scope boundary
In scope:
- wording hierarchy
- pad/body/popup/HUD alignment
- simple role framing
- buy-decision timing guidance
- visual/interaction coherence rules

Out of scope:
- new upgrades
- new currencies
- upgrade rebalance
- new tutorial flows
- new UI surfaces
- changing the core four-step arcade loop
