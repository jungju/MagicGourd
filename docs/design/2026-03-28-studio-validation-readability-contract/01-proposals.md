# Proposals

## Candidate A — Prompt Competition and Lane Readability Contract
Define a Studio checklist and tuning rules for how prompts, signs, and lanes should read while moving through the current loop.

Focus:
- when prompts should appear
- when multiple prompts may compete
- where billboard text becomes too noisy
- how danger should feel readable before contact

## Candidate B — Popup Cadence and Reward Noise Contract
Focus specifically on repeated world popups, banner churn, and reward cadence under fast loop play.

Focus:
- fast pumpkin crack loops
- reward/buy/slow message stacking
- popup text length tolerance
- recovery from noisy feedback moments

## Candidate C — Zone-Edge Pressure Clarity Contract
Focus specifically on how the Nolbu route communicates risk before the player is already slowed.

Focus:
- danger edge anticipation
- goblin first-read clarity
- return-path readability after slow
- where route pressure feels fair vs muddy

## Candidate D — Unified Studio Validation Contract
Create one compact contract that covers:
- traversal and lane clarity
- prompt competition
- billboard readability
- popup overlap
- goblin danger readability

This keeps the remaining blocked work in one place and gives PM/Dev one shared verification artifact instead of fragmented micro-passes.
