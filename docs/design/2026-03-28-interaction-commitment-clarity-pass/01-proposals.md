# Proposals

## Proposal A — Interaction Commitment Hierarchy
Define a single authored hierarchy for when HUD button, local prompt, and object-followthrough should dominate the player's read.

Focus:
- seed button demotion rules
- prompt commitment distance semantics
- unfinished local-object priority
- recovery-state interaction discipline

## Proposal B — Plaza-Only Commitment Polish
Narrow scope to the plaza only.

Focus:
- buy pad spacing
- billboard readability
- prompt capture feel
- post-buy release back into loop

## Proposal C — Mobile/Touch Input Cleanup
Treat the main problem as device-specific input clutter.

Focus:
- button visibility on touch
- prompt size/readability
- avoiding thumb competition with bottom CTA

## Proposal D — Recovery Corridor Interaction Pass
Focus only on danger-edge recovery and re-entry.

Focus:
- slow-state button behavior
- retreat-side prompt noise
- restoring one safe local action after pressure

## Proposal Compression
Highest-value candidates for current refinement mode:
1. Proposal A — covers the whole loop without expanding mechanics
2. Proposal C — important, but best treated as a validation lens under A rather than the main design
3. Proposal D — useful but narrower; already partially covered by prior danger/re-entry work

Proposal B is too narrow for the remaining completion gap.
