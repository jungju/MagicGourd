# Candidates

## Candidate A — Whole-loop interaction commitment hierarchy
Create one small, explicit contract describing:
- which surface owns the moment
- when the seed button becomes secondary/hidden/de-emphasized
- when a local object interaction should outrank plaza spending
- how prompt distance communicates commitment, not just convenience

### Upside
- finishes a real completion gap across the full loop
- aligns code reality with authored design language
- supports later Studio validation without adding systems

### Cost
- requires careful scope discipline so it stays refinement-only

---

## Candidate B — Device-first touch readability pass
Write a touch-oriented polish contract for bottom CTA vs prompts.

### Upside
- helps the blocked validation bucket directly
- likely noticeable on mobile

### Cost
- too implementation-channel-specific for the main remaining design gap
- risks duplicating broader interaction rules later

---

## Candidate C — Post-danger interaction recovery pass
Specify how input surfaces should behave during/after slow recovery.

### Upside
- emotionally important because danger interruptions are memorable
- easy to keep small

### Cost
- narrower than the current whole-loop opportunity
- substantial overlap with the existing re-entry clarity pass

## Candidate Recommendation
Adopt **Candidate A**.

It absorbs the strongest parts of B and C while staying inside the current arcade structure and refinement scope.
