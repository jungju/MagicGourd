# Proposals

## Proposal A — Touch-first input ownership contract
Define explicit ownership rules for when the bottom plant CTA should lead, demote, hide, or wait while touch players enter local prompt spaces.

Why it helps:
- directly addresses the remaining mobile readability blocker
- builds on existing interaction-commitment work instead of replacing it
- stays inside refinement mode

## Proposal B — Plaza-only mobile spend pass
Focus only on touch readability while approaching and buying plaza upgrades.

Why it helps:
- very small scope
- likely to catch one of the most crowded touch slices

Why it may be insufficient:
- ignores village heal and center-lane plant/break competition
- too local for the remaining blocked bucket

## Proposal C — Recovery-state touch clarity pass
Focus on how slowed / recovering players should read the screen on touch when they still have seeds, pumpkins, or nearby prompts.

Why it helps:
- directly targets one high-friction edge case
- strong feel improvement if current touch play feels mushy under pressure

Why it may be insufficient:
- too state-specific
- should probably live inside a broader ownership contract

## Proposal D — Mobile validation checklist only
Add only a touch/mobile validation script without authoring new design rules.

Why it helps:
- safest possible pass
- easy PM handoff

Why it may be insufficient:
- the repo already has validation contracts
- without ownership rules, testers can spot issues but still improvise fixes ad hoc
