# Candidates

## Candidate A — Threshold-first spend coherence
Combine Proposal A and D.

Focus:
- one nearest-threshold rule
- one soft recommendation for when to return to plaza
- small copy updates connecting payouts to the next buy

Why it matters:
- closes the gap between `earned money` and `know what that means`
- improves both reward feel and route clarity
- stays inside the current HUD/popup structure

## Candidate B — Pumpkin-count progress phrasing
Focus on translating cost gaps into harvest-count language.

Why it matters:
- makes the economy feel tactile
- uses the game's core action unit instead of abstract money math

Risk:
- can become too clever if overused
- some thresholds become awkward once upgrade costs diverge

## Candidate C — Multi-affordable plaza choice emphasis
Focus on how the UI should behave once several upgrades are affordable at once.

Why it matters:
- important for mid-run spend decisions
- helps the plaza feel less like three equal text rows

Risk:
- overlaps with existing plaza role/readability passes
- lower value if the player still lacks strong threshold anticipation before arriving
