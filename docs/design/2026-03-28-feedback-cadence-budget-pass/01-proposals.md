# Proposals

## Refine Targets Collected
1. Pumpkin crack progress can visually pile up next to payout in repeated loops.
2. Plaza purchase payoff is now spatially anchored, but its cadence relative to recent payout is not yet authored.
3. Slow-fade recovery is intentionally quiet, but there is no shared rule for when it should yield to a more meaningful reward event.
4. Heal and plant confirmations are both useful, yet their relative intensity at loop speed is still mostly implicit.
5. Banner and world-popup coexistence is defined per event category, but not yet governed by a loop-level rhythm contract.

## Candidate Set

### Candidate A — Popup Lifetime and Suppression Budget
Define duration, yield, and overlap rules so repeated world popups stay legible in fast play.

### Candidate B — Reward Beat Hierarchy Contract
Define which loop events should emotionally dominate when multiple valid feedback beats happen close together.

### Candidate C — Combined Feedback Cadence Budget
Unify priority, suppression, and intensity into one compact rhythm contract for the current loop.

### Candidate D — Purchase-Only Reward Emphasis Pass
Focus narrowly on making plaza spending always read as the strongest non-danger payoff after payout.
