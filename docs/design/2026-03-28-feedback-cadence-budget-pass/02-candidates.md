# Candidates

## Candidate A — Popup Lifetime and Suppression Budget
A narrow tuning contract for popup duration, bucket suppression, and local overlap discipline.

### Strengths
- Cheap to apply
- Directly targets clutter
- Low implementation risk

### Weaknesses
- Too narrow by itself
- Reduces noise but does not fully define reward rhythm

---

## Candidate B — Reward Beat Hierarchy Contract
A priority system for deciding which events deserve the strongest read when several happen close together.

### Strengths
- Improves perceived polish
- Strengthens payout/purchase feel
- Helps danger and reward read distinctly

### Weaknesses
- Can stay abstract without concrete suppression rules
- Needs a timing companion to become actionable

---

## Candidate C — Combined Feedback Cadence Budget
A single contract covering event priority, short suppression windows, yield rules, and intended feel per loop beat.

### Strengths
- Solves the actual remaining gap end-to-end
- Helps Studio validation for popup overlap noise
- Keeps scope small and presentation-only
- Works with existing banner/world-popup architecture

### Weaknesses
- Slightly more authored than a pure timing tweak
- Needs disciplined wording to avoid over-specifying implementation

---

## Candidate D — Purchase-Only Reward Emphasis Pass
A targeted pass that makes plaza buys feel more distinct after farming payout.

### Strengths
- Good emotional payoff
- Very small scope

### Weaknesses
- Leaves heal/plant/crack cadence unresolved
- Too narrow to close the remaining blocked readability concern
