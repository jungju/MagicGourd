# Proposals

## Proposal A — Feedback Anchor Consistency Pass
Define a strict rule for which events get:
- top banner only
- world popup only
- both surfaces
- no popup

Also define the physical origin for each popup so rewards read as coming from the object or zone that caused them.

### Upside
- Directly improves placement consistency
- Cheap to implement inside current remote payload flow
- Clarifies reward cause/effect

### Downside
- Mostly presentation polish; does not improve routing or balance by itself

---

## Proposal B — Event Intensity Ladder
Keep current feedback surfaces, but author a stronger emotional hierarchy:
- routine confirmation
- progress update
- payout moment
- warning / danger
- recovery

### Upside
- Improves reward feel and pacing
- Helps avoid samey reactions across different events

### Downside
- If done alone, placement inconsistency remains

---

## Proposal C — Upgrade Presence Feedback Cleanup
Focus only on plaza upgrades and make purchase / cannot-afford / maxed states read better in both world and HUD surfaces.

### Upside
- Tight and small
- Fixes the clearest current omission: upgrades have banner text but no world celebration

### Downside
- Too narrow for the highest-value whole-loop refinement

---

## Proposal D — Feedback Overlap and Cooldown Rules
Define what happens when multiple feedback events fire close together, including overwrite, suppression, and tone precedence.

### Upside
- Prevents clutter as players farm quickly
- Helps long-run readability

### Downside
- Valuable, but best authored after the surface/anchor map is settled
